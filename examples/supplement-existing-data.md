# 示例：已有数据补充采集

用户输入示例：

```text
这些文件是客户给的历史资产清单，请用 red-ics 继续联网补充采集、交叉核验，并生成最新报告。
```

预期执行模式：`supplement`

执行要点：

- 读取用户提供的资产文件和已有 `output/` 产物。
- 对已有组织、域名、子域名、IP、CIDR、Web、供应商、云服务、APP、小程序、公众号和泄露线索继续联网补充采集。
- 对已有资产做归属核验、状态归一化和可信度标记。
- 如果用户已明确主动授权范围，继续执行主动探测；否则先完成被动联网补充，并在报告中列出待确认的主动范围。
- 生成更新后的 `Report.md`、`assets_summary.xlsx` 和 `evidence_index.csv`。

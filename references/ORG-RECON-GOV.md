# ORG-RECON-GOV — 组织侦察（政府/事业单位专项版）

> **目标**：从单一政府单位或事业单位名称出发，构建目标组织的完整行政关系图谱，
> 挖掘其上下级单位、直属机构、挂靠单位及数字资产。
> **适用场景**：有完整授权的政府/事业单位红蓝对抗、HW行动前期侦察。
>
> **与企业版的关键差异**：
> - 无股权穿透 → 改用行政架构穿透（上下级/直属/挂靠）
> - 域名体系以 `.gov.cn` 为核心，备案审批独立于企业 ICP
> - 云服务偏向政务云/自建机房，较少使用商业云
> - 需增加公安备案筛查、政务服务/公共平台资产发现等专项

## 一、行政架构穿透分析

### 1.1 数据源

| 平台 | 数据范围 | 查询方式 |
|------|---------|---------|
| 中央机构编制网 (scopsr.gov.cn) | 中央/国务院直属机构编制 | 免费 |
| 地方政府网站 (各省市政府官网) | 地方机构设置/直属单位 | 免费 |
| 事业单位在线 (gjsy.gov.cn) | 事业单位登记信息 | 免费，按单位名称查询 |
| 国家企业信用信息公示系统 | 政府下属企业/国企 | 免费 |
| 政府采购网 (ccgp.gov.cn) | 采购单位/中标方/预算信息 | 免费 |
| 公共资源交易平台 | 招标公告 → 单位内部系统线索 | 免费 |

### 1.1.1 政府/事业单位侦察硬性规则

- 信息收集必须联网执行，优先使用编制网、事业单位在线、政府官网、信息公开、采购招标、备案、证书透明日志、搜索引擎和已配置 MCP 数据源交叉验证。
- 先确认行政区划、上级主管单位、机构类型和名称变体，再扩展直属单位、内设机构、下属事业单位、下属企业和信息化供应商。
- 政府/事业单位资产不能只依赖 `.gov.cn` 判断归属；必须结合备案主体、官网引用、采购公告、证书字段、IP Whois 或政务云/运营商归属证据。
- 阶段 1 产出的组织、域名、供应商、政务云、IP/CIDR 线索必须写入后续 OSINT 阶段的种子文件或日志，不能只写在报告正文里。

### 1.1.2 IPSearch-MCP / ip.db 硬性规则

- 目标包含 IPv4 或单位名称时，必须尝试使用 IPSearch-MCP 补充政府出口 IP 段、政务云 CIDR、运营商政企专线等线索。
- 依赖检查顺序固定为：检查已配置 MCP Server → 在本机常见工具目录、当前仓库、`scripts/`、用户工具目录中查找 `ipsearch-mcp` 和 `ip.db` → 本地仍不存在时尝试从 GitHub 下载或按项目说明安装 → 下载/安装失败则记录原因并跳过，继续下一阶段。
- `ipsearch-mcp` 可运行时，必须通过 AI MCP 工具调用 `ip_lookup` 或 `keyword_lookup`；禁止自己写 Python 直接读取或解析 `ip.db`。
- 需要生成关键词计划时，运行 `python scripts/ipsearch_mcp_plan.py --unit-name "<单位名称>"`；查询 IPv4 时运行 `python scripts/ipsearch_mcp_plan.py --ipv4 <IPv4>`。该脚本只生成 MCP 调用计划，不执行数据库查询。
- 单位名称查询必须完整执行所有关键词规则，即使前一个关键词已有结果也不能提前停止；禁止使用 `zhejiang`、`gov`、`hospital` 这类单独宽泛关键词。
- 返回结果必须统一为 `CIDR | IP段描述/名称原文 | 中文归属解释`，并逐条说明该 IP 段归属目标、上级主管单位或政务云平台的原因。

### 1.2 行政架构穿透层级

```
第1层: 目标单位本身
  → 单位全称、简称、曾用名、俗称
  → 统一社会信用代码（事业单位法人证书号）
  → 单位地址、办公地址
  → 单位职能范围（获取业务关键词 → 系统类型推测）
  → 行政级别（正部/副部/正厅/副厅/正处/副处/正科/副科）

第2层: 上级主管单位
  → 直属上级单位（如: 市教育局 → 市政府 / 省教育厅）
  → 业务指导单位（如: 市疾控中心 → 市卫健委）
  → 上级单位的其他下属单位 → 同一上级下的兄弟单位

第3层: 下属/直属单位
  → 直属事业单位（如: 市XX局下属XX中心、XX所、XX站、XX院）
  → 派出机构（如: XX分局、XX办事处、XX管理部）
  → 挂靠单位（如: XX办公室挂靠XX局）
  → 内设机构（如: 办公室、人事处、财务处、信息中心）

第4层: 关联单位
  → 同一办公地址的其他单位
  → 同一业务口的兄弟单位（如: 同属卫健委系统的医院/疾控/监督所）
  → 同一信息化承建方的其他单位
  → 合署办公单位（一套人马两块牌子）

第5层: 下属企业/国企
  → 政府出资成立的平台公司
  → 事业单位下属的经营性企业
  → 后勤服务公司 / 机关服务中心
```

### 1.3 关键提取字段

```
从每个关联实体提取:
  □ 单位全称（含曾用名、俗称、简称）
  □ 统一社会信用代码 / 事业单位法人证书号
  □ 单位负责人（法人/负责人）
  □ 行政级别
  □ 单位地址（办公地址 vs 注册地址）
  □ 联系电话 / 邮箱
  □ 官方网站域名（gov.cn / .政务.cn / org.cn）
  □ 单位职能描述（关键词 → 系统类型推断）
  □ 内设机构/科室名称 → 可能的子域名（bgs/科室名.xxx.gov.cn）
  □ IP / CIDR 线索 → IPSearch-MCP、IP Whois、政务云、运营商政企段交叉验证
  □ IP段归属证据 → 原始描述、中文翻译、命中关键词、行政区划/单位名称关联原因
  □ 政务云 / 运营商 / 自建机房线索 → 用于判断出口、托管和云服务归属
  □ 经费来源（全额拨款/差额拨款/自收自支）
```

### 1.4 部门/职能名称收集

```
来源:
  □ 单位官网 → "机构设置""内设机构""直属单位""联系我们"
  □ 政府信息公开 → 机构职能、权责清单
  □ 政务服务网 → 该单位提供的在线服务事项
  □ 政府采购网 → 采购公告中提到的系统名称
  □ 招聘公告 → 岗位设置 → 内部科室名称
  □ 新闻/工作动态 → 提到的内部系统、平台

收集内容:
  □ 内设处室/科室名称 → 可能的子域名
  □ 业务系统名称 → 可能的独立域名
  □ 公共服务平台 → 可能的子域名或独立站点
  □ 信息化承建方 → 供应链攻击面
```

## 二、ICP 备案查询（政府/事业单位）

### 2.1 备案查询渠道

```
官方渠道（优先）:
  □ ICP/IP地址/域名信息备案管理系统
    https://beian.miit.gov.cn
    查询路径: 公共查询 → 备案信息查询 → 输入主办单位名称
    注意: 主办单位性质选择 "政府机关" / "事业单位"

爱企查（备选）:
  □ https://aiqicha.baidu.com
    搜索单位名称 → 查看"网站备案"栏目

第三方平台:
  □ 站长之家 ICP查询: https://icp.chinaz.com
    → 按主办单位名称 / 域名 / 备案号查询
  □ 备案查询网: https://beianx.cn
    → 支持按主体名批量查询
  □ ICP备案批量查询: https://icp.aizhan.com
    → 输入单位名批量获取备案域名列表
```

### 2.2 .gov.cn 域名体系

| 域名类型 | 格式 | 说明 |
|---------|------|------|
| 政府网站 | `.gov.cn` | 政府机构专用域名，需审批 |
| 政务中文域名 | `.政务.cn` / `.政务` | 中文域名，与 gov.cn 对应 |
| 公益中文域名 | `.公益.cn` / `.公益` | 事业单位/社会组织专用 |
| 事业单位 | `.org.cn` / `.cn` | 部分事业单位使用 |
| 政府邮箱 | `@xxx.gov.cn` | 政府邮箱域名 |

### 2.3 gov.cn 域名特点

```
审批机构: 中央网信办 / 各级编办
注册限制: 仅限政府机关单位申请，企业不得注册

特点:
  □ .gov.cn 域名指向的 IP 通常不起 CDN（或仅用政务云 CDN）
  □ 多数 gov.cn 子域名可直接解析到源站 IP
  □ 同一单位下的子域名安全性参差不齐（主站安全，子站薄弱）
  □ 泛解析型 gov.cn → 可以按科室名批量猜测子域名
  □ gov.cn 域名的子域名是独立站点，可能由不同外包商开发
```

### 2.4 备案信息反查

```
通过域名反查备案主体:
  → 确认 gov.cn 是否真的属于目标单位
  → 发现挂在该主体名下但"看起来不像"的域名

通过备案号反查:
  → 同一备案号下的所有域名 = 同一主办单位
  → 同一主体可能有多个备案号（不同年份办理）

通过单位名称模糊查询:
  → 用单位简称、曾用名补充查询
  → 可能发现历史遗留的未变更域名
```

## 三、公安备案筛查

### 3.1 公安备案说明

```
法规依据: 《计算机信息网络国际联网安全保护管理办法》
要求: 网站在开通30日内到公安机关办理备案

备案号格式: 省简称 + 公网安备 + 数字 + 号
示例: 京公网安备 1101050200XXXX号

关键特征:
  □ 公安备案有独立编号，不同于ICP备案号
  □ 公安备案的网站名称可能与ICP备案不同
  □ 公安备案URL可能包含非标准端口或路径
  □ 政府/事业单位网站必须完成公安备案（硬性要求）
  □ 未完成公安备案的网站 = 合规风险 = 可能是被遗忘的资产
```

### 3.2 查询方法

```
数据源:
  □ 全国公安机关互联网安全管理服务平台: http://www.beian.gov.cn
  □ 网站底部显示的公安备案图标和编号（点击可跳转备案详情）
  □ 第三方备案查询平台（icp.chinaz.com 也支持公安备案查询）

查询策略:
  1. 批量检查: 对所有子域名检查页面底部是否有公安备案号
  2. 编号提取:
     grep -oP '[京津沪渝冀豫云辽黑湘皖鲁新苏浙赣鄂桂甘晋蒙陕吉闽贵粤青川宁琼]公网安备\s*\d+\s*号'
  3. 编号关联: 同一公安备案编号 → 同一主体下的网站
  4. 网站名称关联: 同一公安备案网站名称 → ICP备案关系核实

重要说明:
  □ 公安备案必须有网安图标+备案号链接
  □ 有些子域名可能共用主站公安备案号 → 属于同一运营主体
  □ 不同公安备案号 → 可能代表不同分支机构独立运营
  □ 公安备案的网站名称可能揭示内部系统（如 "XX局OA系统"）
```

## 四、上级/下级单位域名穷举

### 4.1 行政体系域名规律

```
gov.cn 域名常见命名模式:
  □ 行政区划 + 单位:     beijing.gov.cn / shanghai.gov.cn
  □ 行政区划缩写 + 单位:  bjrd.gov.cn（北京人大）
  □ 单位拼音全称:        guangdongshengtongjiju.gov.cn（广东省统计局）
  □ 单位拼音缩写:        stats.gov.cn（国家统计局）
  □ 行业类别:           moe.gov.cn（教育部）/ mohrss.gov.cn（人社部）

子域名常见命名:
  □ 科室拼音:   bgs.xxx.gov.cn（办公室）/ rsc.xxx.gov.cn（人事处）
  □ 系统功能:   xxgk.xxx.gov.cn（信息公开）/ zwfw.xxx.gov.cn（政务服务）
  □ 业务平台:   zfws.xxx.gov.cn（政务服务）/ jszb.xxx.gov.cn（招投标）
  □ 邮件系统:   mail.xxx.gov.cn / email.xxx.gov.cn
  □ 信息公开:   xxgk.xxx.gov.cn / zwgk.xxx.gov.cn / zfxxgk.xxx.gov.cn
  □ 政务服务:   zwfw.xxx.gov.cn / wsbs.xxx.gov.cn / bsdt.xxx.gov.cn
  □ 互动交流:   hdjl.xxx.gov.cn / zxzx.xxx.gov.cn
```

### 4.2 批量域名生成脚本

```python
#!/usr/bin/env python3
"""政府/事业单位子域名字典生成"""

# 常见政府机构科室/部门拼音缩写
gov_depts = [
    "bgs",      # 办公室
    "rsc",      # 人事处
    "cwc",      # 财务处
    "jck",      # 监察科
    "xfb",      # 信访办
    "xxzx",     # 信息中心
    "dzj",      # 电子政务
    "fgw",      # 发改委/法规处
    "zcfg",     # 政策法规处
    "ghc",      # 规划处
    "jdc",      # 监督处
    "jyj",      # 教育局
    "wjw",      # 卫健委
    "sfj",      # 司法局
    "gaj",      # 公安局
    "zjj",      # 住建局
    "jtj",      # 交通局
    "slj",      # 水利局
    "swj",      # 商务局
    "wglj",     # 文旅局
]

# 常见网站功能子域名
web_services = [
    "xxgk",     # 信息公开
    "zwgk",     # 政务公开
    "zfxxgk",   # 政府信息公开
    "zwfw",     # 政务服务
    "wsbs",     # 网上办事
    "bsdt",     # 办事大厅
    "hdjl",     # 互动交流
    "zxzx",     # 在线咨询
    "wzgl",     # 网站管理
    "mail",     # 邮件系统
    "smtp",     # 邮件
    "vpn",      # VPN
    "oa",       # 办公系统
    "portal",   # 门户
    "www",      # 主站
    "old",      # 旧站
    "new",      # 新站
    "test",     # 测试站
    "admin",    # 管理后台
]

# 常见公共服务平台子域名
public_services = [
    "zwfw",     # 政务服务平台
    "ggzy",     # 公共资源交易
    "zfcg",     # 政府采购
    "xzsp",     # 行政审批
    "gsgg",     # 公示公告
    "ggtz",     # 公告通知
]

def generate_subdomains(domain):
    """生成候选子域名列表"""
    candidates = []
    for svc in web_services + public_services:
        candidates.append(f"{svc}.{domain}")
    for dept in gov_depts:
        candidates.append(f"{dept}.{domain}")
    return candidates

# 使用示例
domain = "xxx.gov.cn"
for sub in generate_subdomains(domain):
    print(sub)
```

## 五、政府采购 / 招标信息挖掘

### 5.1 数据源

| 平台 | 信息类型 | 查询方式 |
|------|---------|---------|
| 中国政府采购网 (ccgp.gov.cn) | 采购公告/中标公告/合同公告 | 搜采购单位名称 |
| 各省政府采购网 | 地方采购信息 | 搜单位名称 |
| 公共资源交易平台 | 招标/中标/流标公告 | 搜招标人/采购人 |
| 中国政府购买服务信息平台 | 服务采购 | 搜购买主体 |
| 千里马招标网 (qianlima.com) | 企业+政府招标 | 搜单位名 |
| 剑鱼标讯 (jianyu360.com) | 招标中标 | 搜单位名 |

### 5.2 从采购公告提取的信息

```
采购公告中可能暴露:
  □ 单位内部系统名称:       "XX局信息化系统升级项目"
  □ 现有系统架构:           "基于XX平台/XX数据库/XX中间件"
  □ 信息化承建方:           "由XX公司承建" / "中标供应商: XX科技有限公司"
  □ 预算金额:               → 系统规模评估（100万以下为小系统，500万以上为核心系统）
  □ 联系人 + 电话:          → 社工入口
  □ 技术要求文档:           → 详细技术栈、架构图
  □ 运维服务:               → 驻场人员信息（部分公开在采购需求中）

提取关键词:
  □ 系统名称: 管理系统/平台/大数据/云平台/数据中心/政务云
  □ 技术栈: Java/.NET/Oracle/MySQL/Hadoop/Spring/达梦/人大金仓
  □ 供应商: 中标方/承建方/运维方 → 联系方式和公司背景
```

### 5.3 招标信息关联链

```
链条:
  采购公告 → 中标方 → 该供应商中标的其他项目 → 其他政府单位
  → 同一供应商服务多家单位 → 相同的技术栈/系统架构
  → 该供应商的源码仓库/知识库 → 可能含所有客户的配置

操作:
  1. 搜索目标单位所有采购公告（近3年）
  2. 提取每个项目的中标方 → 供应商清单
  3. 搜索每个供应商中标的其他项目 → 关联单位
  4. 搜索供应商公开的源码仓库（GitHub/Gitee）→ 单位配置泄露
  5. 特别关注 "运维服务" 类采购 → 供应商可能持有系统最高权限
```

## 七、政务云与信息化建设识别

### 7.1 政务云识别

```
政府/事业单位偏向使用政务云而非商业云:

常见政务云:
  □ 省级政务云（各省自建，如 浙江政务云、广东政务云）
  □ 浪潮云 (inspur.com) → 服务于大量政府客户
  □ 华为政务云 (Huawei Cloud Stack)
  □ 阿里云电子政务云 (aliyungovernment.com)
  □ 中国电信政务云 / 天翼云政务云
  □ 中国联通政务云 / 沃云
  □ 曙光政务云
  □ 紫光政务云

识别方法:
  □ DNS CNAME 记录 → *.govcloud.xxx / *.chinagovermentcloud.com
  □ IP 归属查询 → 运营商段（电信/联通/移动）
  □ 政府采购公告 → "政务云服务采购""云资源租赁"
  □ 网站底部 → "技术支持: XX政务云"
```

### 7.2 自建机房识别

```
政府单位常见自建/托管模式:
  □ 机房托管在运营商 IDC（电信/联通/移动）
  □ 信息中心自建机房
  □ 各级大数据局统一管理
  □ 电子政务外网 / 电子政务内网

自建机房特征:
  □ IP 归属: 中国电信/中国联通/中国移动 政企客户段
  □ 非云服务商 AS 号（如 AS4134/AS4837/AS9808）
  □ 同一 IP 段下有多个同类 gov.cn 站点
  □ 网站底部无云服务商标识
```

## 八、扩展资产专项

### 8.1 政府新媒体矩阵

```
必查平台:

  □ 微博官方号 → 链接/二维码可能指向独立系统
  □ 政务抖音号 → 主页链接
  □ 政务头条号
  □ 政府 App（政务服务/便民服务） → 在应用商店搜索单位名
  □ 微信小程序 / 支付宝小程序（政务服务类）

提取信息:
  □ 新媒体账号名称 → 可能的子域名
  □ 服务入口跳转的 URL
  □ 外包开发公司线索（小程序/APP开发方）
  □ 便民服务中的在线系统入口
```

### 8.2 事业单位下属企业

```
政务体系中的经营性实体:
  □ 机关服务中心 → 后勤/物业公司（独立网站）
  □ 信息中心下属公司 → 信息化运维/开发（可能有源码仓库）
  □ 培训中心/会议中心 → 独立网站（通常无gov.cn）
  □ 出版社/报社 → 独立域名（.com / .cn）
  □ 研究院/设计院 → 独立域名
  □ 招待所/宾馆 → 独立网站

这些下属企业:
  □ 通常不使用 .gov.cn
  □ 使用 .com / .cn / .com.cn / .org
  □ 安全水平远低于政府主站
  □ 可能与主站共享同一内网/机房/云账号
```

### 8.3 信息公开中的系统线索

```
政府信息公开栏目中的线索:
  □ 财政预决算 → "信息化项目"支出 → 系统建设/运维合同
  □ 政府采购 → 系统建设合同详情 → 承建方+技术方案
  □ 规划计划 → 信息化规划 → 未来系统清单+预算
  □ 人事信息 → 信息中心人员 → 可能的技术栈偏好（招聘岗位要求）
  □ 统计公报 → 数字化建设数据 → 系统运行规模
  □ 年度报告 → 信息化建设成果 → 已上线系统清单
```

## 十、与企业组织侦察的关键差异总结

| 维度 | 企业版 (ORG-RECON) | 政府/事业单位版 (ORG-RECON-GOV) |
|------|-------------------|-------------------------------|
| 组织关系 | 股权穿透（天眼查/企查查） | 行政架构穿透（编制网/政府网站） |
| 域名体系 | .com/.cn 为主 | .gov.cn/.政务.cn 为主 |
| ICP备案 | beian.miit.gov.cn | 同上 + 政府域名审批系统 |
| 公安备案 | 可选筛查 | 必查（硬性要求） |
| CDN 状况 | 大量使用商业 CDN | 较少 CDN，多数直接解析源站 |
| 云服务 | 阿里云/腾讯云/AWS 等 | 政务云/自建机房为主 |
| 招标信息 | 商业招标平台 | 政府采购网 + 公共资源交易 |
| 扩展资产 | 小程序/APP/电商 | 政务服务/信息公开/新媒体矩阵 |
| 资产关联 | 股权→子公司→备案域名 | 行政层级→直属单位→独立子站 |
| 信息源 | 天眼查/招聘/年报 | 采购公告/信息公开/编制网 |
